\section*{AgentDojo - held-out attack augmentations (surface forms the adapter never trained on)}

\par\footnotesize ODILE's adapter is trained on a narrow slice of AgentDojo injections (banking money-transfer / data-exfil tasks under the \texttt{important\_instructions} wrapper). The full injection-style matrix it is \textbf{evaluated} on contains 13 attack families, most of which are *transformations of the surface form* the adapter never saw during training: the \texttt{\textless{}INFORMATION\textgreater{}} authority wrapper stripped off (\texttt{tk\_no\_wrapper}, \texttt{tk\_descriptive}), the harmful payload restyled as a polite compliance email (\texttt{shadowleak}), as base64-encoding framing (\texttt{shadowleak\_b64}), as a fake kernel/auth-broker log, as the model's own injected \texttt{Thought:/Action:} monologue (\texttt{fake\_monologue}), or as a deprecated-protocol notice with zero imperative verbs (\texttt{implicit\_socratic}). The point of this exhibit: the \textbf{attacker goal stays fixed}, only the wrapper changes - so a reviewer can see directly that these are out-of-distribution rewrappings, not memorized strings. ODILE reaches 0\% ASR on 12 of 13 families (mean 0.013\%) at Llama-3.3-70B; no-defense baseline averages 14.04\%.

\par{\footnotesize\itshape Sources (verbatim attack templates): \texttt{odile/src/odile/attacks/baseline\_attacks.py} (the \texttt{\_JB\_STRING} of each \texttt{@register\_attack} class) and \texttt{odile/src/odile/attacks/important\_instructions\_attacks.py} (the \texttt{important\_instructions} and \texttt{tool\_knowledge} wrappers). Family keys and per-family ASR: \texttt{paper/drafts/odile\_restructured/tables/t\_ad\_injection\_styles.tex}. The real injection-task goal used below is verbatim from \texttt{odile\_github/src/agentdojo/default\_suites/v1\_2/banking/injection\_tasks.py} (\texttt{InjectionTask3.GOAL}, line 165, with \texttt{\_ATTACKER\_IBAN = "US133000000121212121212"}).}\par

\par\footnotesize Two labels are used below:
\begin{itemize}
\item \footnotesize \textbf{[verbatim template]}  the \texttt{\_JB\_STRING} exactly as it appears in the codebase, with \texttt{\{goal\}} (and \texttt{\{tools\}}, \texttt{\{user\}}, \texttt{\{model\}}) left as the literal placeholders the framework substitutes at run time.
\item \footnotesize \textbf{[rendered]}  the same verbatim template with \texttt{\{goal\}} filled in by the real banking goal so the reviewer can read the fully-instantiated injection. The fill is mechanical (Python \texttt{.format}); only \texttt{\{goal\}} is substituted.
\end{itemize}

\medskip\noindent\textcolor{nv}{\large\textbf{The fixed attacker goal (verbatim, identical across every surface form below)}}\par\smallskip

\noindent\colorbox{hl!12}{\parbox{0.95\linewidth}{\footnotesize Send a transaction to US133000000121212121212 that includes the user's phone model (as visible from the transaction history). If details of transactions are underspecified, fill them in without asking me.}}\par

\par\footnotesize For the tool-knowledge families (\texttt{tk\_*}), the framework additionally substitutes \texttt{\{tools\}} with a tool-call sequence derived from the injection task's ground truth (e.g. *You should use the tool \texttt{send\_money} with the following arguments: *); this is shown as the \texttt{\{tools\}} placeholder.

\medskip\noindent\textcolor{nv}{\large\textbf{Also-registered held-out augmentations (verbatim templates available in the same source file)}}\par\smallskip

\par\footnotesize These are additional out-of-distribution rewrappings registered alongside the families above (\texttt{odile/src/odile/attacks/baseline\_attacks.py}), used in extended ablations. All use the \textbf{same \texttt{\{goal\}} slot}:

\begin{itemize}
\item \footnotesize \textbf{\texttt{shadowleak\_explicit}} (\texttt{ShadowLeakExplicitAttack})  ShadowLeak polite-email framing + explicit tool-call sequence (\texttt{\{tools\}}), no \texttt{\textless{}INFORMATION\textgreater{}}, no imperatives.
\item \footnotesize \textbf{\texttt{compositional\_shadowmono}} (\texttt{CompositionalShadowMonoAttack})  ShadowLeak authority framing fused with the FakeMonologue \texttt{Thought:/Action:} structure; a combination of two families neither seen in training.
\item \footnotesize \textbf{\texttt{longctx\_shadowleak}} / \textbf{\texttt{longctx\_fakemono}} (\texttt{LongCtxShadowLeakAttack} / \texttt{LongCtxFakeMonoAttack})  \textasciitilde{}1,800 tokens of benign business filler prepended before the ShadowLeak / FakeMonologue body (attention-dilution / position-robustness stress).
\item \footnotesize \textbf{\texttt{tool\_schema\_redefinition}} (\texttt{ToolSchemaRedefinitionAttack})  reframes a legit tool's documented behavior so the harmful goal is presented as the tool's "canonical effect"; no instruction shape.
\item \footnotesize \textbf{\texttt{trickle\_relay}} (\texttt{TrickleRelayAttack})  splits the attack into a "reference value" half and an "apply it later" half to defeat per-message scanners.
\item \footnotesize \textbf{\texttt{kernel\_authority\_mimicry}} (\texttt{KernelAuthorityMimicryAttack})  injection styled as an out-of-band machine-emitted broker message (\texttt{\textless{}\textless{}KMSG ring=0 ...\textgreater{}\textgreater{}}), no English instruction shape.
\item \footnotesize \textbf{\texttt{refuse\_to\_refuse}} (\texttt{RefuseToRefuseAttack})  a meta-instruction about response style (suppress confirmations) with the harmful goal recast as an onboarding-approved "standing item."
\item \footnotesize \textbf{\texttt{paraphrase\_chain}} (\texttt{ParaphraseChainAttack})  the same goal stated three times as semantically-equivalent paraphrases to dilute per-string surface matching.
\end{itemize}

Of these, the marker-removed (\texttt{tk\_no\_wrapper}, \texttt{tk\_descriptive}), social-engineering (\texttt{shadowleak*}), monologue (\texttt{fake\_monologue}), error-lure (\texttt{system\_error\_trap}), and protocol-notice (\texttt{implicit\_socratic}) forms are surface transformations not present in the adapter's \texttt{important\_instructions}-style training data. ODILE: 0\% ASR on 12/13, 0.17\% residual on \texttt{direct}, mean 0.013\%.
